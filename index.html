about

My path into GRC wasn't a straight line, and that's what makes it useful. I started in eDiscovery supporting litigation data operations for Merck and Johnson & Johnson, where I learned how organizations manage sensitive data under legal scrutiny. From there I moved through a data center role at Nasdaq and network operations at Verizon Wireless, building a foundation in infrastructure and systems that most compliance professionals never have.

My first compliance exposure came at Syntax, a SaaS company navigating PCI-DSS and SOC I/II audits. Being part of the team that helped prepare for those assessments showed me how much manual effort goes into evidence collection, and how much of it shouldn't be manual at all. That's what pulled me fully into the GRC engineering space.

Since then I've worked primarily in the NIST and FedRAMP compliance space across federal, DoD, and healthcare environments. My current role has expanded that into SOC I/II, HITRUST, and PCI-DSS. My goal is to make continuous compliance the default and build the tooling that lets organizations walk into any audit confident their controls will hold.

projects

project_01
Serverless IAM Access Review System
Orchestrates IAM, IAM Access Analyzer, and Security Hub findings (IAM.1–IAM.9) into an AI-narrated HTML compliance report. EventBridge triggers a Lambda pipeline weekly and deposits a dated self-contained report to S3. Deployed via CloudFormation.
python aws lambda iam security hub eventbridge cloudformation bedrock s3

project_02
Compliant S3 Primitive — NIST 800-53 as Terraform
Terraform module enforcing five NIST 800-53 controls — SC-28, AC-3, CM-6, AU-3, AU-6 — directly in infrastructure. After deploy, terraform show -json produces machine-readable compliance evidence an auditor can traverse without a console login or a screenshot.
terraform aws s3 kms nist 800-53 sc-28 iac-as-evidence hipaa
▶ walkthrough github →
project_03
Compliant GCS Bucket — NIST 800-171 as Terraform (GCP)
Reusable GCP Terraform module that hard-codes six NIST 800-171 controls — SC-12, SC-13, SC-28, AC-3, CM-6, AU-11 — into every Cloud Storage bucket it provisions. Plan-time validation gates reject non-compliant config before any resource reaches GCP. After apply, a compliance_attestation output surfaces machine-readable evidence of every enforced control.
terraform gcp cloud kms nist 800-171 cmek sc-28 iac-as-evidence
▶ walkthrough github →
project_04
GKE Cluster Compliant Primitive — Multi-Framework
Ephemeral, FedRAMP-realistic GKE cluster with private nodes, Cloud NAT egress, and control plane locked to operator /32. Application-layer secrets encrypted via Cloud KMS with 90-day rotation. Four NIST 800-53 controls hard-coded into the cluster definition. Evidence lives in artifacts/plan.json and artifacts/terraform-state.json — auditor-traversable without a console login.
terraform gcp gke kubernetes nist 800-53 nist 800-171 fedramp cmmc l2 soc 2 kms
